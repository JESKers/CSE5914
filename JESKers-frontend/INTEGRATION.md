# Integration Guide ‚Äî plugging the backend into the frontend

This frontend is the **display layer**. It is finished and runs standalone on
mock data. This doc is the contract for connecting the other three pieces. The
whole integration happens through **one file**: [`src/api/client.js`](src/api/client.js).

[`src/types.js`](src/types.js): `CarRecord`, `SearchRequest`, `SearchResponse`.

## The contract (must not drift)

Request the frontend sends:

```jsonc
// SearchRequest
{
  "query": "corvette",                 // free-text keywords
  "filters": {
    "makes": ["Chevrolet"],
    "model": "corvette",
    "engineTypes": ["premium unleaded (required)"],
    "cylinders": [8],
    "transmissions": ["AUTOMATED_MANUAL"],
    "drivenWheels": ["rear wheel drive"],
    "minPrice": 30000, "maxPrice": 60000,
    "minHp": 350, "maxHp": null,
    "minYear": 2022, "maxYear": null
  },
  "sort": "price-asc",                 // relevance | price-asc | price-desc | hp-desc
  "page": 1, "size": 9
}
```

Response the backend must return:

```jsonc
// SearchResponse
{
  "results": [ /* CarRecord[] ‚Äî see src/types.js / mockCars.js */ ],
  "total": 42, "page": 1, "size": 9
}
```

## Who does what

### Kangjie ‚Äî Elasticsearch
1. Create the `cars` index. Make `make`/`model` both `text` (full-text) **and**
   `keyword` (exact filter/aggregation) via multi-field; `year`/`msrp`/`engine.hp`
   numeric (range); `transmission_type`/`driven_wheels`/`engine.fuel_type`
   `keyword` (terms).
2. Import the Kaggle CSV so each document matches `CarRecord` (clean N/A ‚Üí `[]`,
   empty hp ‚Üí `null`).
3. The query shape your index must answer is generated by `buildEsQuery()` in
   `src/api/client.js` ‚Äî copy it as the reference.

### Eric ‚Äî Integration (Python API)
1. Expose `POST /api/search` taking a `SearchRequest`, returning a `SearchResponse`.
2. Easiest path: take the body from `buildEsQuery()` (or rebuild it server-side),
   send to ES `_search`, map `hits.hits[]._source` ‚Üí `results`, `hits.total.value`
   ‚Üí `total`.
3. Tell the frontend to use it (see "Go live" below).

### Jerry ‚Äî RAG LLM (Timebox 3, later)
The same `searchCars()` call carries `query`. In TB3, route `query` through the
RAG pipeline instead of (or alongside) keyword search and return the same
`SearchResponse`. No frontend change.

## Go live (no code edits)

In `frontend/`:

```bash
cp .env.example .env
# .env:
VITE_API_BASE=/api
VITE_PROXY_TARGET=http://localhost:8000   # where Eric's API runs
```

Restart `npm run dev`. The client auto-switches off mock as soon as
`VITE_API_BASE` is set. To force either mode, set `VITE_USE_MOCK=true|false`.

## Verify it's wired

- Search returns the same cars whether mock or live (sanity check a few).
- Filters narrow results; pagination + sort work against real totals.
- Network tab shows `POST /api/search` returning a `SearchResponse`.
- Removing all filters returns the full dataset count.
